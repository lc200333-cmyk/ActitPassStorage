# ActitPassStorage

Современный локальный менеджер секретов в духе SPB Wallet.

В текущем состоянии репозитория есть:

- `plans/` - подробные планы продукта и реализации.
- `web/` - запускаемое приложение без внешних зависимостей с зашифрованными локальными базами.
- `tools/dev-server.js` - маленький локальный статический сервер и API синхронизации.
- `tests/` - тесты Node для ядра vault и sync API.
- `app/` - Flutter-приложение для production-сборок.
- `core/` - Rust workspace с доменными типами vault/sync и FFI-точкой входа.

## Как запустить

Запускаемая версия использует browser storage и WebCrypto, поэтому работает без Flutter и Rust SDK:

```bash
npm run dev
```

Затем откройте:

```text
http://127.0.0.1:4173
```

## Docker-сборки APK/deb/setup.exe

Основной способ сборки APK и deb не требует установки Flutter, Android SDK или Rust на хост: нужен только Docker и `docker-compose`.

Подробная инструкция: [docs/builds.md](docs/builds.md).

Собрать Docker-образ с окружением:

```bash
npm run docker:build-image
```

Собрать Android APK:

```bash
npm run docker:apk
```

Собрать Linux deb:

```bash
npm run docker:deb

sudo apt install --reinstall ./dist/actit-pass-storage_0.1.0_amd64.deb
```

`docker:deb` собирает пакет в Ubuntu 20.04 контейнере, чтобы избежать ошибки `GLIBC_2.34 not found` на старых Linux.

Собрать APK и deb подряд:

```bash
npm run docker:release
```

Проверить проект внутри контейнера:

```bash
npm run docker:test
```

Все результаты складываются в `dist/`, которая подключена в контейнер как volume:

```text
dist/ActitPassStorage-android-debug.apk
dist/actit-pass-storage_0.1.0_amd64.deb
```

Контейнерное окружение описано в:

```text
docker/build-env/Dockerfile
docker/linux-deb/Dockerfile
docker-compose.yml
```

Windows `setup.exe` собирается на Windows runner через GitHub Actions, потому что Flutter Windows desktop требует Windows SDK/MSVC и не кросс-собирается в Linux Docker:

```text
.github/workflows/windows_setup.yml
```

Ожидаемый артефакт workflow:

```text
ActitPassStorage-Setup-0.1.0.exe
```

```bash

```

Android APK:

```bash
npm run build:apk
```

Ожидаемый артефакт:

```text
dist/ActitPassStorage-android-debug.apk
```

Linux deb:

```bash
npm run build:deb
```

Ожидаемый артефакт:

```text
dist/actit-pass-storage_0.1.0_amd64.deb
```

Важно: Flutter Windows desktop собирается на Windows-хосте. На Linux-хосте локально собираются APK и deb после установки Flutter, Android SDK, Rust и Linux desktop dependencies.

## Реализовано в запускаемой версии

- Создание и открытие зашифрованных баз по мастер-паролю.
- При создании база просит повторить мастер-пароль и может сразу добавить русские демо-карточки.
- Все поля ввода пароля, включая мастер-пароль и пароли синхронизации, имеют кнопку-глаз для раскрытия.
- Русские встроенные шаблоны: пароль, защищенная заметка, банковская карта, документ, банковский счет, лицензия ПО, доступ к серверу, Wi-Fi.
- Пиктограммы для шаблонов и выбор пиктограммы в редакторе.
- Отдельная витрина всех пиктограмм в настройках.
- Настраиваемые шаблоны и поля.
- Цвета карточек с видимыми цветными кружками.
- Отдельная витрина всей палитры цветов в настройках.
- Секретные поля скрыты по умолчанию, есть раскрытие кнопкой-глазом и копирование.
- В готовых шаблонах скрываются именно пароли и CVV; номера карт, счетов, документов и лицензий показываются обычным текстом.
- Карточки показывают все заполненные поля: обычные поля открыто, скрытые поля с глазом.
- Генератор паролей и TOTP-коды.
- Теги, категории, поиск и сортировка.
- Зашифрованный экспорт/импорт базы.
- Зашифрованные пакеты синхронизации с правилом “последнее изменение побеждает”.
- Повторный импорт пакетов синхронизации не создает дубликаты конфликтов.
- Синхронизация через локальную/сетевую папку, WebDAV и FTP.
- Формы настройки и инструкции в интерфейсе для папки/SMB/NFS, почты, WebDAV, SFTP и FTP/FTPS.
- Журнал конфликтов с отметкой просмотра.
- Автоблокировка и очистка буфера обмена.
- PWA manifest и offline service worker.

## Примеры настройки синхронизации

Папка / SMB / NFS:

1. Создайте папку для пакетов синхронизации.
2. Для SMB/NFS сначала подключите ресурс средствами системы.
3. В приложении выберите `Папка / SMB / NFS` и укажите путь:

```text
/home/user/ActitPassSync
```

WebDAV:

1. Создайте папку в Nextcloud, ownCloud, NAS или другом WebDAV-хранилище.
2. Укажите URL папки:

```text
https://example.com/remote.php/dav/files/user/ActitPass/
```

3. Если нужна авторизация, заполните пользователя и пароль приложения в форме или используйте JSON:

```json
{"url":"https://example.com/remote.php/dav/files/user/ActitPass/","username":"user","password":"app-password"}
```

FTP/FTPS:

1. Создайте папку на FTP-сервере.
2. Введите хост, порт, логин, пароль и путь.
3. Обычный FTP передает учетные данные без защиты; данные базы остаются зашифрованными, но лучше использовать WebDAV через HTTPS или SFTP.

```text
ftp://user:password@example.com/ActitPass/
```

Почта IMAP/SMTP:

1. Подготовьте почтовый ящик или отдельную папку/метку.
2. Укажите IMAP сервер для чтения и SMTP сервер для отправки.
3. Используйте пароль приложения.
4. Запускаемая версия умеет автоматический обмен пакетами через IMAP/SMTP: приложение отправляет зашифрованный sync-пакет письмом самому себе и читает такие письма обратно.
5. Поддерживаются обычное TCP-подключение и implicit TLS на типовых портах 993/465. STARTTLS на 587 и OAuth-провайдеры остаются задачей промышленного ядра.

SFTP:

1. Подготовьте SSH/SFTP доступ к серверу.
2. Создайте удаленную папку для пакетов синхронизации.
3. Укажите хост, порт, пользователя и путь.
4. В текущей запускаемой версии форма готова, автоматический SFTP адаптер относится к промышленному этапу.

## Целевая промышленная версия

Промышленная цель остается Flutter + Rust:

- Flutter UI в `app/`.
- Rust ядро vault/sync в `core/`.
- `flutter_rust_bridge` как мост между приложением и ядром.

Web-реализация остается рабочим эталоном UX и поведения, пока Flutter/Rust слой догоняет ее по глубине криптографии и провайдерам.
